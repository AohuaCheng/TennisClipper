# 数据隐私政策

> 本文档明确 Tennis Rally Clipper 项目的数据处理原则，保障所有贡献者的隐私权益。

---

## 核心原则

> **不默认公开任何球友的原始视频。**

本项目是**本地优先**工具，所有视频处理默认在本地完成，不强制上传任何数据至云端或 GitHub。

---

## 数据分类与处理

| 数据类型 | 存储位置 | 处理方式 | 是否可公开 |
|---|---|---|---|
| **原始打球视频** | 本地硬盘 | 仅本地分析，不上传 | ❌ 否 |
| **代理视频 (proxy)** | 本地硬盘 | 低分辨率分析用，不上传 | ❌ 否 |
| **标注 CSV** | 本地硬盘 | 脱敏后可选共享结构 | ⚠️ 需授权 |
| **脱敏特征 JSON** | 本地 / 可选共享 | 仅包含结构化数字特征，无画面 | ✅ 可以 |
| **模型训练脚本** | GitHub 仓库 | 代码 | ✅ 可以 |
| **合成 Demo 视频** | 本地 / 可选共享 | 需获得视频中所有人物授权 | ✅ 可以 |
| **剪辑结果** | 本地 | 用户自行决定是否发布 | 用户决定 |

---

## 使用 B 站 / 小红书 / 第三方视频

- 仅作为**私人参考**，了解目标剪辑结果和标注标准
- **不用于**开源训练集分发
- **不放入** GitHub 仓库
- **不公开**引用或展示

---

## 贡献数据的正确方式

如果你想为项目提供数据支持：

### 1. 提供原始视频（最严格）

- 确保你拥有视频的完整使用权
- 确保视频中所有人物知情并同意用于研究
- 视频仅由核心维护者本地使用，不分享
- 不公开视频的任何帧或片段

### 2. 提供脱敏特征（推荐）

这是**最推荐**的贡献方式：

- 你在本地运行 Tennis Rally Clipper 的分析流程
- 导出**结构化特征 JSON**（仅包含时间、坐标、运动数值等）
- 我们验证 JSON 格式正确性，不查看任何画面
- 这些特征可用于训练模型，完全不包含视频内容

示例脱敏特征：

```json
{
  "t": 120.0,
  "motion_energy_total": 0.63,
  "motion_energy_near_player": 0.72,
  "player_near_x": 591,
  "player_near_y": 800,
  "ball_detected": false
}
```

### 3. 提供授权 Demo 视频

- 视频中的所有人明确同意公开
- 视频仅用于项目演示（如 README 中的示例 GIF）
- 片段时长不超过 30 秒

## 数据删除权

如果你曾经贡献过数据，有权随时要求：
- 删除你提供的原始视频
- 删除对应的标注记录
- 从训练集中移除相关数据

请通过 GitHub Issue 联系维护者，我们将在 7 天内处理。

---

## 技术层面的隐私保护

### 本地优先架构

- 视频分析在本地运行
- 不依赖云端 API（除非用户明确选择 Phase 5 的 LLM 精筛）
- 代理视频降低分析成本，无需上传原视频

### 不收集遥测

- 不收集用户行为数据
- 不收集崩溃报告（除非用户主动提交）
- 不追踪使用情况
